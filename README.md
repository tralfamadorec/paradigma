# Нисходящее проектирование. Процедурная парадигма. Git.  
  
## Общая постановка задачи  
  
Необходимо реализовать консольное приложение, которое имеет примитивный текстовый интерфейс и меню, состоящее как минимум из четырех пунктов:  
- Ввод исходных данных, как вручную, так и сгенерированных случайным образом  
- Выполнение алгоритма по заданию  
- Вывод результата  
- Завершение работы программы  
  
---
  
### Особые указания:  
  
- Алгоритм не может быть выполнен без введенных данных  
- Результат не может быть выведен без обработки введённых данных  
- Вывод результата не может быть осуществлен без выполнения алгоритма  
- При вводе новых данных результаты выполнения алгоритма "сбрасываются"  
  
Разработка ведётся в концепции нисходящего проектирования.  
  
---
  
## Конкретная постановка задачи  
  
Входные данные: целочисленная матрица.  
Результат работы алгоритма: максимальный среди минимальных элементов, располагающихся на диагоналях, параллельных побочной диагонали матрицы.  
  
---
  
## Структура меню  
  
Основное меню содержит следующие пункты:  

1\. Ввод исходных данных  
&nbsp;  - Пользователь выбирает способ ввода:  
&nbsp;    - 1.1. Вручную: ввод размера квадратной матрицы и её элементов построчно.  
&nbsp;    - 1.2. Случайная генерация: указание размера и диапазона значений; матрица генерируется автоматически.  

2\. Выполнение алгоритма  
&nbsp;  - Выполняет обработку введённой матрицы в соответствии с вариантом.  
&nbsp;  - Доступен только после ввода данных.  

3\. Вывод результата  
&nbsp;  - Выводит вычисленное значение (целое число).  
&nbsp;  - Доступен только после выполнения алгоритма.  

4\. Завершение работы  
&nbsp;  - Корректный выход из программы.  
  
---
  
## Принципы реализации  
  
- Программа разрабатывается методом нисходящего проектирования.  
- На начальных итерациях функции реализуются как заглушки.  
- Реальная логика ввода данных и алгоритма добавляется постепенно.  
- Обработка состояний (проверка наличия данных и результата) реализована в главной функции.
